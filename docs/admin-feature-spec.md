# 오늘도수영 앱 기능 명세서

작성일: 2026-06-24

## 1. 앱 개요

오늘도수영 앱은 수영장 회원의 고정 강습 일정, 담당 강사, 잔여 횟수권, 결석, 빈자리 신청을 관리하는 앱입니다. 관리자는 회원 정보, 회원권 상품, 고정수업, 날짜별 수업 운영 상태를 캘린더에서 확인하고 수정할 수 있습니다.

## 2. 사용자 구분

### 회원

- 본인의 다음 수업 일시와 담당 강사를 확인합니다.
- 본인의 고정수업 목록과 남은 횟수권을 확인합니다.
- 고정수업에 결석 처리할 수 있습니다.
- 결석으로 열린 빈자리에 추가 수업을 신청하거나 취소할 수 있습니다.
- 다음 고정수업 1회에 한해 수업 변경 요청을 보낼 수 있습니다.
- 공지사항을 확인합니다.
- 프로필에서 휴대폰 번호, 앱 버전, 계정 삭제 메뉴를 확인합니다.

### 관리자

- 회원 목록을 검색하고 회원별 상세 정보를 확인합니다.
- 회원의 잔여 횟수권을 추가하거나 차감합니다.
- 회원의 결제 상품을 1:1, 1:2, 1:3 중 하나로 설정합니다.
- 회원별 고정수업 요일, 시간, 담당 강사를 등록, 수정, 취소합니다.
- 운영 캘린더에서 날짜별 수업을 열고 닫습니다.
- 운영 캘린더에서 수업 담당 강사와 수업 길이를 수정합니다.
- 운영 캘린더에서 고정수업 외 개별 수업을 회원에게 직접 배정하거나 배정 취소합니다.
- 회원이 보낸 수업 변경 요청을 승인하거나 거절합니다.
- 공지사항을 작성하고 이미지를 첨부할 수 있습니다.

## 3. 회원가입 및 로그인

- 회원가입은 이메일, 비밀번호, 이름, 휴대폰 번호를 입력해 진행합니다.
- 회원가입한 계정은 기본적으로 일반 회원으로 생성됩니다.
- 관리자 권한은 Supabase 관리자 화면 또는 DB에서 별도로 부여해야 합니다.
- 휴대폰 번호는 회원 정보에 저장되며 관리자 화면에서 확인할 수 있습니다.
- 로그인과 계정 관리는 Supabase Auth를 사용합니다.

## 4. 회원권 및 결제 상품 관리

### 회원권 기본 구조

- 회원은 현재 활성 회원권을 1개 가집니다.
- 회원권에는 다음 정보가 저장됩니다.
  - 결제 상품: 1:1, 1:2, 1:3
  - 총 횟수
  - 남은 횟수
  - 활성 여부
  - 구매일

### 기본값

- 신규 가입 회원은 기본 8회권으로 생성됩니다.
- 신규 가입 회원의 기본 상품은 1:1입니다.
- 관리자가 회원 상세 화면에서 상품을 1:1, 1:2, 1:3으로 변경할 수 있습니다.

### 횟수권 처리

- 회원이 추가 수업 또는 빈자리 수업을 예약하면 남은 횟수가 1회 차감됩니다.
- 예약을 취소하거나 관리자가 배정을 취소하면 남은 횟수가 1회 복구됩니다.
- 관리자는 회원 상세 화면에서 남은 횟수를 수동으로 +1, -1 조정할 수 있습니다.
- 모든 횟수 변동은 거래 이력으로 저장됩니다.

## 5. 상품별 수업 인원 제한 규칙

수업별로 1:1, 1:2, 1:3을 따로 저장하지 않습니다. 대신 해당 수업에 들어가는 회원들의 결제 상품을 기준으로 최대 인원이 결정됩니다.

운영 규칙:

- 1:1 회원이 들어간 수업에는 다른 회원을 추가 배정할 수 없습니다.
- 1:2 회원이 들어간 수업은 최대 2명까지만 배정할 수 있습니다.
- 1:3 회원들만 들어간 수업은 최대 3명까지 배정할 수 있습니다.
- 서로 다른 상품 회원이 섞이면 더 작은 상품 기준을 따릅니다.
  - 예: 1:2 회원과 1:3 회원이 같은 수업에 있으면 최대 2명까지만 가능합니다.

이 제한은 앱 화면뿐 아니라 DB 함수에서 강제로 검사합니다. 따라서 관리자가 캘린더에서 직접 배정하더라도 상품 기준을 넘는 배정은 저장되지 않습니다.

## 6. 고정수업 관리

고정수업은 회원이 매주 반복해서 듣는 기본 강습 시간입니다.

관리자는 회원 상세 화면에서 다음 정보를 등록합니다.

- 회원
- 요일
- 시작 시간
- 담당 강사

고정수업 시작 시간은 정시와 30분 단위를 지원합니다.

- 예: 13:00
- 예: 13:30

현재 지원하는 시작 분 단위는 00분과 30분입니다.

관리자는 회원별로 여러 개의 고정수업을 등록할 수 있습니다. 고정수업을 수정하면 이후 일정과 관련된 결석/대체 예약도 규칙에 맞게 정리됩니다.

## 7. 운영 캘린더

관리자 홈에는 운영 캘린더가 표시됩니다.

캘린더에서 확인할 수 있는 정보:

- 날짜
- 시작 시간
- 수업 열림/닫힘 상태
- 담당 강사
- 수업 길이
- 고정 회원
- 개별 배정 회원
- 결석 회원
- 빈자리 여부

캘린더 편집 기능:

- 수업 열기
- 수업 닫기
- 담당 강사 변경
- 수업 길이 변경
- 선택한 회원을 특정 수업에 개별 배정
- 선택한 회원의 개별 배정 취소

수업 길이는 다음 두 가지를 지원합니다.

- 30분
- 1시간

수업을 닫으면 해당 수업의 대체 예약 또는 개별 배정은 취소되고, 차감된 횟수권은 복구됩니다.

## 8. 결석 및 빈자리 예약

### 회원 결석 처리

- 회원은 본인의 고정수업에 결석 처리할 수 있습니다.
- 결석 처리된 수업은 빈자리로 공개됩니다.
- 이미 다른 회원이 해당 빈자리를 예약한 경우, 결석 취소가 제한될 수 있습니다.

### 빈자리 예약

- 다른 회원은 공개된 빈자리에 추가 수업을 신청할 수 있습니다.
- 신청 시 남은 횟수권이 1회 차감됩니다.
- 신청을 취소하면 횟수권이 1회 복구됩니다.
- 상품별 인원 제한을 초과하는 신청은 저장되지 않습니다.

## 9. 다음 수업 변경 요청

회원은 다음 고정수업 1회에 대해 변경 요청을 보낼 수 있습니다.

요청 조건:

- 변경하려는 대상 수업은 빈자리가 있어야 합니다.
- 본인의 기존 고정수업 시간으로는 요청할 수 없습니다.
- 이미 같은 수업에 신청되어 있으면 요청할 수 없습니다.
- 상품별 인원 제한을 만족해야 합니다.

관리자 처리:

- 관리자는 요청 목록에서 승인 또는 거절할 수 있습니다.
- 승인하면 기존 다음 고정수업은 결석 처리되고, 요청한 수업에 회원이 배정됩니다.
- 이 변경은 추가 수업 예약이 아니므로 별도 횟수권 차감 없이 처리됩니다.

## 10. 공지사항

- 관리자는 공지사항을 작성할 수 있습니다.
- 제목, 내용, 이미지를 등록할 수 있습니다.
- 회원과 관리자는 공지사항 목록을 확인할 수 있습니다.
- 공지 이미지는 Supabase Storage에 저장됩니다.

## 11. 알림 설정

앱에는 로컬 알림 설정 화면이 있습니다.

설정 가능한 항목:

- 수업 리마인드
- 예약 관련 알림
- 공지사항 알림
- 출석 관련 알림
- 수업 변경 알림
- 재예약 관련 알림

현재 알림은 앱 내 로컬 알림 중심으로 구현되어 있습니다.

## 12. 계정 삭제

- 회원은 프로필 화면에서 계정 삭제를 요청할 수 있습니다.
- 계정 삭제는 Supabase Edge Function을 통해 처리됩니다.
- 앱에는 DB 비밀번호나 service role key를 저장하지 않습니다.

## 13. 보안 및 데이터 관리

이 앱은 Supabase Auth와 Supabase Postgres를 중앙 DB로 사용합니다.

보안 원칙:

- 앱에는 DB 비밀번호를 넣지 않습니다.
- 앱에는 Supabase publishable key만 사용합니다.
- 민감한 예약 처리, 횟수권 차감/복구, 인원 제한 검사는 DB 함수에서 처리합니다.
- RLS 정책으로 회원은 자기 데이터만 보고, 관리자는 운영 데이터에 접근할 수 있게 분리합니다.
- 계정 삭제처럼 강한 권한이 필요한 작업은 Edge Function에서 처리합니다.

주요 데이터:

- 회원 정보
- 회원권 상품 및 잔여 횟수
- 고정수업
- 날짜별 수업 슬롯
- 결석 정보
- 빈자리 예약 및 개별 배정
- 수업 변경 요청
- 횟수권 거래 이력
- 공지사항

## 14. 현재 운영상 유의사항

- 회원가입 직후에는 일반 회원입니다. 관리자 권한은 별도로 부여해야 합니다.
- 신규 회원의 기본 상품은 1:1, 기본 횟수는 8회입니다.
- 수업 시작 시간은 현재 00분 또는 30분 단위로 입력합니다.
- 결제 자체를 앱에서 직접 처리하는 기능은 아직 구현되어 있지 않습니다.
- 실제 결제 내역 입력은 관리자가 회원권 상품과 잔여 횟수를 수동으로 관리하는 방식입니다.
- 최신 DB 구조를 사용하려면 Supabase SQL Editor에서 최신 `supabase/schema.sql`을 실행해야 합니다.
