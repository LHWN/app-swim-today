# 오늘도수영 앱 시연 순서 및 기능 명세서

작성일: 2026-07-09

## 1. 시연 목표

오늘도수영 앱은 수영장 회원의 수업 일정 확인, 결석/변경/추가수업/자유수영 신청, 특별수업 신청, 상품 구매, 문의 접수를 한 앱에서 처리하고, 관리자는 회원권, 고정수업, 운영 수업, 승인 요청, 공지, 피드백, 상품 판매를 관리하는 모바일 앱입니다.

시연에서는 다음 흐름을 보여주는 것이 핵심입니다.

- 회원은 홈에서 내 수업과 요청 상태를 빠르게 확인한다.
- 회원은 수업 탭에서 결석, 변경, 추가수업, 자유수영을 신청한다.
- 관리자는 홈에서 들어온 요청을 승인하거나 거절한다.
- 관리자는 회원권, 고정수업, 날짜별 수업 운영, 공지, 피드백, 특별수업, 상품 판매를 관리한다.
- 회원은 처리 결과, 공지, 피드백, 구매 신청, 문의 접수를 앱 안에서 확인한다.

## 2. 시연 전 준비

시연 전에 아래 상태를 준비해두면 전체 흐름이 자연스럽습니다.

- 관리자 계정 1개와 회원 계정 1개 이상을 준비한다.
- 회원에게 고정수업 1개 이상을 배정해둔다.
- 오늘 이후 날짜에 열린 수업과 빈자리 수업이 보이도록 데이터가 있어야 한다.
- 결석 요청, 변경 요청, 추가수업 신청, 자유수영 신청 중 최소 1개는 새로 만들 수 있는 상태가 좋다.
- 특별수업 1개, 상품 1개, 공지 1개를 미리 등록해두면 시연 시간이 줄어든다.
- 개발 모드에서는 `.env`의 테스트 계정 전환 버튼을 활용하면 회원/관리자 전환이 빠르다.

## 3. 추천 시연 순서

### 3.1 로그인 및 역할 구분

1. 앱을 실행하고 로그인 화면을 보여준다.
2. 회원 계정으로 로그인한다.
3. 하단 탭 구조를 간단히 설명한다.
   - 홈
   - 수업
   - 공지
   - 알림
   - 내정보

말할 포인트:

- 앱은 회원과 관리자 계정의 화면이 분리된다.
- 회원가입 계정은 기본적으로 회원이며, 관리자 권한은 별도로 부여한다.
- Supabase Auth와 DB 권한 정책을 사용한다.

### 3.2 회원 홈 대시보드

1. 홈 화면 상단의 다음 예약 수업을 보여준다.
2. 남은 수업권, 내 고정수업 요약, 관리자 확인 대기 건수를 설명한다.
3. 메뉴 버튼을 순서대로 훑는다.
   - 내 수업
   - 요청 관리
   - 특별수업
   - 상품 구매
   - 수업 피드백
4. 내 수업에서 예정 수업과 지난 내 수업 진입을 보여준다.

말할 포인트:

- 고정수업, 추가로 신청한 수업, 자유수영이 시간 순서대로 정리된다.
- 요청 관리와 상품 구매에는 도움말 툴팁이 있어 화면 설명을 접어둘 수 있다.
- 지난 수업에서는 날짜, 강사, 시간이 한 줄로 정리된다.

### 3.3 회원 수업 탭

1. 수업 탭으로 이동한다.
2. 날짜 선택 영역과 남은 수업권 요약을 보여준다.
3. 내 고정수업 카드에서 결석 요청 버튼을 보여준다.
4. 결석 요청을 만든다.
5. 같은 고정수업에서 변경 버튼을 눌러 빈자리 선택 흐름을 보여준다.
6. 빈자리 또는 열린 수업에서 추가수업 신청을 보여준다.
7. 닫힌 시간의 자유수영 신청 가능 상태를 보여준다.

말할 포인트:

- 결석 요청은 관리자 승인 후 빈자리로 공개된다.
- 결석 요청은 수업 전날까지만 가능하다.
- 회원권 8회당 결석 요청은 2회까지만 가능하다.
- 변경 요청은 원래 고정수업을 다른 빈자리로 옮기는 요청이며, 관리자 승인 후 반영된다.
- 추가수업은 남은 수업권이 있어야 신청할 수 있다.

### 3.4 관리자 홈: 승인 요청 처리

1. 관리자 계정으로 전환하거나 재로그인한다.
2. 홈 화면의 운영 요약을 보여준다.
   - 회원 수
   - 고정수업 수
   - 빈자리 수
   - 대체 배정 수
3. 승인 요청 메뉴에서 들어온 요청을 확인한다.
   - 결석 요청
   - 수업/자유수영 신청
   - 변경 요청
   - 회원 문의/요구사항
4. 회원이 만든 결석 요청을 승인한다.
5. 승인 후 해당 수업이 빈자리로 공개되는 흐름을 설명한다.

말할 포인트:

- 관리자는 홈에서 처리해야 할 요청을 한곳에서 본다.
- 승인/거절 결과는 요청 상태로 남는다.
- 횟수권 차감, 복구, 인원 제한은 DB 함수에서 트랜잭션으로 처리된다.

### 3.5 관리자 회원 관리

1. 회원 관리 메뉴로 이동한다.
2. 이름, 이메일, 전화번호 검색을 보여준다.
3. 회원 상세에서 아래 정보를 보여준다.
   - 이름, 이메일, 휴대폰
   - 회원권 상품 1:1, 1:2, 1:3
   - 남은 횟수권
   - 고정수업 요약
4. +1, -1로 횟수권 조정을 보여준다.
5. 수업 버튼을 눌러 고정수업 등록/수정/취소 화면을 보여준다.

말할 포인트:

- 같은 시간에 여러 회원이 들어갈 수 있지만, 1:1, 1:2, 1:3 상품 인원 규칙을 넘을 수 없다.
- 고정수업을 수정하거나 취소하면 이후 일정과 관련된 결석/대체 상태도 DB 규칙에 따라 정리된다.

### 3.6 관리자 수업 탭: 날짜별 운영 관리

1. 수업 탭으로 이동한다.
2. 날짜 선택 후 하루 수업 운영표를 보여준다.
3. 수업 열기, 수업 닫기, 강사 변경, 수업 길이 변경을 설명한다.
4. 특정 시간에 회원을 직접 배정하거나 배정 취소할 수 있음을 보여준다.
5. 이미 지난 수업도 최근 2주치까지 확인 가능하다고 설명한다.
6. 수업 탭에서 피드백 작성 버튼을 보여준다.

말할 포인트:

- 운영자는 날짜별 실제 수업 슬롯을 기준으로 운영 상태를 관리한다.
- 고정수업, 결석, 대체 배정, 빈자리 상태를 한 카드에서 확인한다.
- 피드백은 종료된 수업의 회원별 기록으로 남는다.

### 3.7 콘텐츠: 특별수업, 피드백, 공지

1. 관리자 홈의 콘텐츠 메뉴를 보여준다.
2. 특별수업 등록 또는 수정 화면을 보여준다.
   - 제목
   - 설명
   - 이미지
   - 일시
   - 강사
   - 수업 시간
   - 정원
3. 특별수업 신청자 승인/거절을 설명한다.
4. 수업 피드백 대상 목록과 피드백 작성 화면을 보여준다.
   - 사진
   - 동영상
   - 100자 피드백
5. 공지 탭으로 이동해 공지 목록을 보여준다.
6. 관리자 계정에서 글쓰기, 수정, 삭제를 보여준다.
7. 회원 계정에서 공지를 게시판처럼 조회하는 모습을 보여준다.

말할 포인트:

- 공지는 관리자만 작성, 수정, 삭제할 수 있고 회원은 조회 중심이다.
- 피드백 미디어는 앱 안에서 이미지와 동영상을 확인하는 흐름을 목표로 한다.
- 특별수업은 신청, 대기, 승인, 거절 상태를 가진다.

### 3.8 상품 구매

1. 회원 홈의 상품 구매 메뉴로 이동한다.
2. 상품 목록과 구매 신청 버튼을 보여준다.
3. 구매 신청 후 내 구매 신청 화면에서 상태를 확인한다.
4. 관리자 홈의 상품 판매 메뉴로 이동한다.
5. 상품 등록과 구매 신청 확정/취소를 보여준다.

말할 포인트:

- 상품 등록은 관리자만 가능하다.
- 회원 구매는 먼저 신청 상태가 되고, 관리자가 확정하거나 취소한다.
- 취소 시 재고가 복구된다.

### 3.9 내정보, 문의, 알림

1. 회원 내정보 탭으로 이동한다.
2. 기본 정보 메뉴를 눌러 이메일, 계정 유형, 휴대폰, 남은 횟수권을 보여준다.
3. 문의 접수 메뉴에서 문의를 작성한다.
4. 전화 문의 버튼을 보여준다.
5. 알림 탭에서 알림 항목별 토글을 보여준다.

말할 포인트:

- 회원의 불편사항/요구사항은 내정보의 문의 접수에서 작성한다.
- 관리자는 홈의 승인 요청에서 문의를 처리중, 해결, 거절 상태로 관리한다.
- 알림은 수업, 예약, 공지, 출석, 변경, 재예약 항목으로 나뉜다.

### 3.10 마무리

마지막에는 아래 세 가지를 요약한다.

- 회원은 수업 신청과 변경, 결석, 구매, 문의를 모바일에서 처리한다.
- 관리자는 회원권, 고정수업, 수업 운영, 요청 승인, 공지, 피드백, 상품 판매를 한곳에서 처리한다.
- 중요한 예약/횟수권/인원 제한은 앱 화면뿐 아니라 DB 함수에서도 막는다.

## 4. 기능 명세

### 4.1 사용자 역할

#### 회원

- 본인의 다음 수업과 예정 수업을 확인한다.
- 본인의 고정수업, 추가수업, 자유수영 일정을 확인한다.
- 지난 내 수업과 수업 피드백을 확인한다.
- 수업 탭에서 결석 요청, 수업 변경 요청, 추가수업 신청, 자유수영 신청을 한다.
- 요청 상태를 확인하고 대기 중인 요청을 취소할 수 있다.
- 특별수업에 신청하거나 신청을 취소할 수 있다.
- 상품을 구매 신청하고 대기 중인 구매 신청을 취소할 수 있다.
- 내정보에서 기본 정보, 문의 접수, 전화 문의, 정책 문서를 확인한다.
- 알림 항목별 수신 여부를 설정한다.

#### 관리자

- 전체 회원과 운영 현황을 확인한다.
- 결석, 변경, 추가수업, 자유수영, 문의 요청을 승인/거절/처리한다.
- 회원 검색, 회원권 상품 변경, 횟수권 조정, 고정수업 등록/수정/취소를 처리한다.
- 강사별 수업 요일과 시간을 관리한다.
- 날짜별 수업 슬롯을 열고 닫고, 강사/수업 길이/정원/배정 상태를 관리한다.
- 수업 종료 후 회원별 피드백을 작성한다.
- 특별수업을 등록/수정하고 신청자를 승인/거절한다.
- 상품을 등록하고 구매 신청을 확정/취소한다.
- 공지를 작성, 수정, 삭제한다.

### 4.2 공통 내비게이션

앱의 기본 탭은 다음과 같다.

- 홈: 역할별 대시보드
- 수업: 날짜별 수업 확인 및 신청/운영
- 공지: 공지 목록 및 관리자 글쓰기
- 알림: 알림 설정
- 내정보: 기본 정보, 문의, 전화 문의, 계정 관련 메뉴

### 4.3 회원 홈

회원 홈은 모바일 대시보드 형태로 구성된다.

- 다음 예약 수업 표시
- 남은 수업권 표시
- 내 고정수업 요약 표시
- 관리자 확인 대기 요청 수 표시
- 내 수업, 요청 관리, 특별수업, 상품 구매, 수업 피드백 메뉴 제공

내 수업:

- 앞으로 예정된 고정수업, 추가수업, 자유수영을 시간 순으로 표시한다.
- 지난 내 수업 상세로 이동할 수 있다.

요청 관리:

- 결석 요청, 변경 요청, 추가수업/자유수영 신청 상태를 표시한다.
- 대기 중인 요청은 회원이 취소할 수 있다.
- 설명 문구는 툴팁으로 숨겨 화면 밀도를 줄인다.

특별수업:

- 모집 중인 특별수업 목록을 확인한다.
- 신청, 대기, 승인, 거절 상태를 확인한다.
- 신청 취소가 가능하다.

상품 구매:

- 등록된 상품을 확인한다.
- 상품 구매 신청을 생성한다.
- 내 구매 신청 내역을 확인하고 대기 중 신청을 취소한다.

수업 피드백:

- 관리자가 작성한 수업별 피드백을 확인한다.
- 사진 또는 동영상이 있는 경우 앱 안에서 확인한다.

### 4.4 회원 수업 탭

회원 수업 탭은 선택한 날짜에 회원이 할 수 있는 액션만 노출한다.

- 내 고정수업 표시
- 내 추가수업 또는 자유수영 표시
- 결석으로 생긴 빈자리 표시
- 추가수업 신청 가능 슬롯 표시
- 자유수영 신청 가능 슬롯 표시

결석 요청:

- 본인의 고정수업에만 가능하다.
- 관리자 승인 전에는 승인 대기 상태로 표시된다.
- 대기 중 요청은 회원이 취소할 수 있다.
- 수업 당일에는 결석 요청이 마감된다.
- 활성 회원권 기준 8회당 2회까지만 가능하다.
- 승인 후 해당 자리는 빈자리로 공개된다.

수업 변경 요청:

- 본인의 고정수업을 선택한 뒤 변경할 빈자리를 선택한다.
- 관리자 승인 후 원래 고정수업은 결석 처리되고 대상 빈자리에 배정된다.
- 변경 요청은 추가수업이 아니므로 별도 횟수권을 차감하지 않는다.

추가수업 신청:

- 열린 수업 중 배정되지 않은 시간에 신청할 수 있다.
- 남은 수업권이 있어야 한다.
- 관리자 승인 시 수업권이 차감된다.

자유수영 신청:

- 영업시간 내 닫힌 시간에 신청할 수 있다.
- 관리자 승인 후 배정 상태가 된다.

### 4.5 관리자 홈

관리자 홈은 운영자가 처리해야 할 업무를 섹션별로 보여준다.

운영 요약:

- 회원 수
- 고정수업 수
- 빈자리 수
- 대체 배정 수

승인 요청:

- 결석 요청 승인/거절
- 추가수업/자유수영 신청 승인/거절 및 처리 코멘트
- 수업 변경 요청 승인/거절
- 회원 문의 처리중/해결/거절 및 답변 메모

회원 관리:

- 이름, 이메일, 전화번호 검색
- 회원권 상품 1:1, 1:2, 1:3 변경
- 남은 횟수권 +1, -1 조정
- 고정수업 등록, 수정, 취소
- 회원별 고정수업 요약 확인

강사:

- 강사별 요일/시간 배정 확인
- 강사 수업 시간 추가, 수정, 삭제

콘텐츠:

- 특별수업 등록, 수정
- 특별수업 신청자 승인/거절
- 수업 피드백 작성
- 최근 공지 확인 및 공지 탭 이동

상품 판매:

- 상품명, 설명, 이미지, 가격, 재고 등록
- 구매 신청 확정/취소
- 처리 메모 작성
- 등록 상품 확인

### 4.6 관리자 수업 탭

관리자 수업 탭은 날짜별 운영표 역할을 한다.

- 날짜별 수업 슬롯 확인
- 오늘 이후 수업과 최근 2주 이내 지난 수업 확인
- 수업 열기
- 수업 닫기
- 담당 강사 변경
- 수업 시간 30분/60분 변경
- 정원 변경
- 회원 직접 배정
- 회원 배정 취소
- 고정수업 출석 취소 처리
- 수업 피드백 작성 진입

수업 카드에는 다음 상태가 표시된다.

- 고정 회원
- 결석 회원
- 대체/개별 배정 회원
- 빈자리 수
- 담당 강사
- 수업 길이

### 4.7 공지

회원:

- 공지 목록을 게시판처럼 확인한다.
- 제목, 본문, 작성자, 작성일, 이미지를 확인한다.

관리자:

- 글쓰기 버튼으로 공지 작성 폼에 진입한다.
- 제목, 본문, 이미지를 등록한다.
- 기존 공지를 수정한다.
- 기존 공지를 삭제한다.

### 4.8 내정보 및 문의

기본 정보:

- 이름
- 이메일
- 로그인 방식
- 회원/관리자 권한
- 휴대폰 번호
- 남은 횟수권 또는 운영 정보
- 앱 테스트 버전 정보

문의 접수:

- 회원은 문의 제목과 내용을 작성한다.
- 문의는 관리자 홈의 승인 요청에 표시된다.
- 관리자는 처리중, 해결, 거절 상태와 답변 메모를 남긴다.

전화 문의:

- 전화 문의 버튼으로 `010-4698-3505`에 연결한다.
- 관리자 홈에서는 우측 상단 문의 버튼을 표시하지 않는다.

### 4.9 알림

알림 설정 항목:

- 수업 전 알림
- 예약 알림
- 공지 알림
- 출석 알림
- 수업 변경 알림
- 재예약 알림

현재 앱은 로컬 알림 중심으로 동작하며, 원격 푸시는 운영 백엔드와 플랫폼 설정이 추가로 필요하다.

### 4.10 데이터 및 권한

주요 저장소:

- Supabase Auth: 로그인과 계정 관리
- Supabase Postgres: 회원, 수업, 요청, 회원권, 공지, 상품, 피드백 데이터
- Supabase Storage: 공지 이미지, 피드백 이미지/동영상, 특별수업 이미지, 상품 이미지

주요 테이블:

- `profiles`: 회원 기본 정보, 권한, 남은 횟수권
- `member_passes`: 활성 회원권, 상품 인원, 총 횟수, 남은 횟수
- `fixed_lessons`: 회원별 고정수업
- `lesson_slots`: 날짜별 실제 수업 슬롯
- `lesson_absence_requests`: 결석 요청
- `lesson_absences`: 승인된 결석 처리
- `lesson_change_requests`: 수업 변경 요청
- `lesson_assignment_requests`: 추가수업/자유수영 신청
- `reservations`: 대체/개별 배정
- `lesson_feedbacks`: 수업 피드백
- `special_lessons`, `special_lesson_registrations`: 특별수업과 신청자
- `member_requests`: 문의/요구사항
- `store_products`, `store_orders`: 상품과 구매 신청
- `notices`: 공지사항
- `pass_transactions`: 횟수권 거래 이력

권한 정책:

- 회원은 자기 데이터 중심으로 조회/신청한다.
- 관리자는 운영 데이터와 전체 회원 요약을 조회/처리한다.
- 중요한 예약 처리와 횟수권 처리는 앱에서 직접 테이블을 수정하지 않고 RPC로 처리한다.

### 4.11 핵심 운영 규칙

회원권/수업 인원:

- 회원권 상품은 1:1, 1:2, 1:3을 지원한다.
- 같은 수업에 여러 회원이 배정될 때 가장 작은 상품 인원 제한을 따른다.
- 인원 제한은 DB 함수에서 최종 검증한다.

결석:

- 결석 요청은 본인의 고정수업만 가능하다.
- 결석 요청은 수업 전날까지만 가능하다.
- 활성 회원권 기준 8회당 2회까지만 가능하다.
- 승인 대기 결석과 승인된 결석을 합산해 제한한다.
- 승인된 결석에 다른 회원이 대체 예약을 한 경우 결석 취소가 제한될 수 있다.

추가수업:

- 열린 수업이면서 아직 배정되지 않은 시간에 신청한다.
- 승인 시 회원권 1회가 차감된다.
- 취소 또는 거절 처리 시 필요한 경우 재고/횟수권이 복구된다.

자유수영:

- 05:00-22:00 영업시간 안의 닫힌 시간에 신청한다.
- 관리자 승인 후 배정된다.

변경 요청:

- 원래 고정수업과 변경 대상 빈자리를 함께 지정한다.
- 승인 시 원래 수업은 결석 처리되고 대상 수업에 배정된다.
- 추가 수업이 아니므로 별도 횟수권 차감은 없다.

특별수업:

- 신청은 선착순으로 queue position을 가진다.
- 정원 내 신청은 승인 대상으로, 정원 초과는 대기 상태로 운영할 수 있다.
- 관리자가 최종 승인/거절한다.

상품 구매:

- 회원이 구매 신청하면 재고가 차감된다.
- 관리자가 취소하면 재고가 복구된다.
- 관리자가 확정하면 구매 신청이 확정 상태가 된다.

공지:

- 관리자가 작성/수정/삭제한다.
- 회원은 조회한다.

피드백:

- 관리자는 종료된 수업 대상자에게 피드백을 작성한다.
- 회원은 작성된 피드백과 미디어를 확인한다.

### 4.12 운영 및 배포 유의사항

- Supabase SQL Editor에서 `supabase/schema.sql`을 최신 상태로 실행해야 DB 기능이 맞게 동작한다.
- 앱에는 Supabase service role key를 넣지 않는다.
- 계정 삭제 같은 고권한 작업은 Edge Function으로 처리한다.
- 실제 외부 테스트 배포는 Expo 빌드 또는 TestFlight/Android 내부 테스트 트랙을 사용한다.
- 원격 푸시, 소셜 로그인 운영, 앱스토어 심사는 별도 플랫폼 설정이 필요하다.
