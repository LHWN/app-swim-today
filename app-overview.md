# 오늘도수영 앱 전반 설명서

작성일: 2026-07-09

## 1. 문서 목적

이 문서는 오늘도수영 앱의 전체 구조를 이해하기 위한 기준 문서입니다. 시연 순서가 아니라, 앱이 어떤 문제를 해결하고 어떤 기능, 정책, 권한, 데이터 구조로 구성되어 있는지 설명합니다.

대상 독자는 다음과 같습니다.

- 앱을 처음 테스트하는 사용자
- 수영장 운영자 또는 관리자
- 앱 유지보수 개발자
- Supabase 데이터 구조를 확인해야 하는 담당자

## 2. 앱 개요

오늘도수영 앱은 수영장 회원의 고정수업, 추가수업, 자유수영, 수업 취소, 특별수업, 상품 구매, 문의 접수, 공지 확인을 모바일에서 처리하기 위한 앱입니다.

운영자는 관리자 계정으로 접속해 회원권, 고정수업, 날짜별 수업 운영, 승인 요청, 공지, 특별수업, 수업 피드백, 상품 판매를 관리합니다.

핵심 방향은 다음과 같습니다.

- 회원은 내 수업과 신청 상태를 한 화면에서 확인한다.
- 회원은 수업을 바꾸고 싶을 때 기존 수업을 취소하고 새 수업을 신청한다.
- 관리자는 회원 요청을 승인 또는 거절하고 운영 데이터를 정리한다.
- 예약, 취소, 횟수권 차감/복구, 인원 제한은 DB 함수에서 최종 검증한다.
- 앱에는 Supabase service role key 같은 고권한 비밀값을 넣지 않는다.

## 3. 사용자 역할

### 3.1 회원

회원은 앱에서 다음 작업을 할 수 있습니다.

- 다음 수업 확인
- 예정된 내 수업 확인
- 지난 내 수업 확인
- 남은 수업권 확인
- 고정수업 취소 요청
- 추가수업 신청
- 자유수영 신청
- 승인 대기 중인 요청 취소
- 승인된 개별수업 취소
- 특별수업 신청 및 취소
- 상품 구매 신청 및 취소
- 수업 피드백 확인
- 공지사항 확인
- 문의 접수
- 전화 문의
- 알림 설정
- 기본 정보와 계정 관련 메뉴 확인

### 3.2 관리자

관리자는 앱에서 다음 작업을 할 수 있습니다.

- 전체 운영 현황 확인
- 회원 검색
- 회원권 상품 변경
- 회원 잔여 횟수권 조정
- 회원별 고정수업 등록, 수정, 취소
- 강사별 수업 시간 관리
- 날짜별 수업 열기, 닫기, 수정
- 특정 수업에 회원 직접 배정
- 특정 수업의 회원 배정 취소
- 회원의 고정수업 취소 요청 승인/거절
- 추가수업/자유수영 신청 승인/거절
- 문의 처리
- 특별수업 등록, 수정, 신청자 승인/거절
- 수업 피드백 작성
- 상품 등록, 구매 신청 확정/취소
- 공지 작성, 수정, 삭제

## 4. 화면 구조

앱의 하단 탭은 공통적으로 다음 구조를 사용합니다.

- 홈
- 수업
- 공지
- 알림
- 내정보

회원과 관리자는 같은 탭 구조를 사용하지만, 각 탭에서 보이는 내용과 액션은 역할에 따라 달라집니다.

### 4.1 회원 홈

회원 홈은 회원이 가장 자주 확인하는 정보를 대시보드 형태로 보여줍니다.

표시 정보:

- 다음 예약 수업
- 남은 수업권
- 내 고정수업 요약
- 관리자 확인 대기 요청 수
- 내 수업 메뉴
- 요청 관리 메뉴
- 특별수업 메뉴
- 상품 구매 메뉴
- 수업 피드백 메뉴

내 수업:

- 고정수업, 추가수업, 자유수영을 시간 순서대로 표시합니다.
- 지난 내 수업 화면으로 이동할 수 있습니다.

요청 관리:

- 수업 취소 요청
- 추가수업 신청
- 자유수영 신청
- 기존 변경 요청 이력
- 대기 중 요청 취소
- 최근 처리 결과 확인

현재 정책상 회원 수업 탭에는 신규 변경 요청 버튼을 제공하지 않습니다. 수업을 바꾸려면 기존 수업을 취소하고 새 수업을 신청하는 흐름을 사용합니다.

### 4.2 회원 수업 탭

회원 수업 탭은 날짜별로 회원이 할 수 있는 수업 관련 작업을 보여줍니다.

표시 대상:

- 내 고정수업
- 내 추가수업
- 내 자유수영
- 승인 대기 중인 추가수업/자유수영 신청
- 추가수업 신청 가능한 열린 수업
- 자유수영 신청 가능한 닫힌 시간

버튼 상태:

- 고정수업: `취소`
- 고정수업 취소 요청 대기: `요청취소`
- 승인된 개별수업/추가수업/자유수영: `취소`
- 승인 대기 중인 추가수업/자유수영 신청: `대기취소`
- 신청 가능한 추가수업: `추가신청`
- 신청 가능한 자유수영: `자유수영`
- 당일 또는 지난 수업: `마감`

### 4.3 관리자 홈

관리자 홈은 운영자가 처리해야 하는 업무를 섹션별로 정리합니다.

상단 요약:

- 회원 수
- 고정수업 수
- 빈자리 수
- 대체/개별 배정 수

관리 메뉴:

- 승인 요청
- 회원 관리
- 강사
- 콘텐츠
- 상품 판매

승인 요청:

- 수업 취소 요청
- 추가수업 신청
- 자유수영 신청
- 기존 변경 요청
- 문의/요구사항

회원 관리:

- 회원 검색
- 회원권 상품 변경
- 잔여 횟수권 +1, -1 조정
- 고정수업 등록
- 고정수업 수정
- 고정수업 취소

강사:

- 강사 이름별 수업 시간 확인
- 요일/시간 배정
- 강사 시간 수정 또는 삭제

콘텐츠:

- 특별수업 등록/수정
- 특별수업 신청자 승인/거절
- 수업 피드백 작성
- 최근 공지 확인

상품 판매:

- 상품 등록
- 구매 신청 확인
- 구매 확정
- 구매 취소
- 처리 메모 작성

### 4.4 관리자 수업 탭

관리자 수업 탭은 날짜별 운영표 역할을 합니다.

관리자는 다음 정보를 확인합니다.

- 날짜
- 수업 시작 시간
- 담당 강사
- 수업 길이
- 수업 열림/닫힘 상태
- 고정 회원
- 취소 회원
- 대체/개별 배정 회원
- 남은 빈자리 수

관리자는 다음 작업을 할 수 있습니다.

- 수업 열기
- 수업 닫기
- 강사 변경
- 수업 길이 변경
- 정원 변경
- 회원 직접 배정
- 배정 취소
- 고정수업 출석 취소 처리
- 수업 피드백 작성

관리자 수업 탭에서는 최근 2주 이내 지난 수업도 확인할 수 있습니다.

### 4.5 공지 탭

회원:

- 공지 목록을 게시판처럼 확인합니다.
- 제목, 본문, 작성자, 작성일, 이미지를 확인합니다.

관리자:

- 글쓰기 버튼으로 공지를 작성합니다.
- 제목, 본문, 이미지를 등록합니다.
- 기존 공지를 수정합니다.
- 기존 공지를 삭제합니다.

### 4.6 알림 탭

알림 탭에서는 알림 수신 항목을 설정합니다.

항목:

- 수업 전 알림
- 예약 알림
- 공지 알림
- 출석 알림
- 수업 변경 알림
- 재예약 알림

현재 구현은 로컬 알림 중심입니다. 운영용 원격 푸시는 별도 백엔드와 플랫폼 설정이 필요합니다.

### 4.7 내정보 탭

내정보 탭은 사용자 기본 정보와 문의 관련 기능을 제공합니다.

기본 정보:

- 이름
- 이메일
- 로그인 방식
- 회원/관리자 권한
- 휴대폰 번호
- 남은 횟수권 또는 운영 정보
- 앱 버전 정보

회원 문의:

- 회원은 문의 제목과 내용을 작성합니다.
- 문의는 관리자 홈의 승인 요청에 표시됩니다.
- 관리자는 처리중, 해결, 거절 상태와 답변 메모를 남길 수 있습니다.

전화 문의:

- 전화 문의 버튼은 `010-4698-3505`로 연결합니다.
- 관리자 홈의 우측 상단 문의 버튼은 표시하지 않습니다.

## 5. 운영 정책

### 5.1 계정 정책

- 회원가입한 계정은 기본적으로 회원입니다.
- 관리자 권한은 DB 또는 Supabase 관리 화면에서 별도로 부여합니다.
- 회원은 자기 데이터 중심으로 조회하고 요청합니다.
- 관리자는 전체 운영 데이터에 접근합니다.

### 5.2 회원권 정책

회원권은 `member_passes`를 기준으로 관리합니다.

주요 속성:

- 회원
- 상품 인원: 1:1, 1:2, 1:3
- 총 횟수
- 남은 횟수
- 활성 여부
- 구매일

기본 정책:

- 신규 회원은 기본 회원권을 가집니다.
- 관리자는 회원권 상품을 1:1, 1:2, 1:3 중 하나로 변경할 수 있습니다.
- 관리자는 잔여 횟수권을 +1 또는 -1 조정할 수 있습니다.
- 횟수권 변동은 `pass_transactions`에 기록됩니다.

### 5.3 수업 인원 정책

수업 인원 제한은 회원권 상품을 기준으로 합니다.

- 1:1 회원이 포함된 수업은 최대 1명입니다.
- 1:2 회원이 포함된 수업은 최대 2명입니다.
- 1:3 회원만 있는 수업은 최대 3명입니다.
- 서로 다른 상품 회원이 섞이면 가장 작은 인원 제한을 따릅니다.

예:

- 1:2 회원과 1:3 회원이 같은 수업에 들어가면 최대 2명까지만 가능합니다.
- 1:1 회원이 있는 수업에는 다른 회원을 추가 배정할 수 없습니다.

이 제한은 앱 화면뿐 아니라 DB 함수에서 최종 검사합니다.

### 5.4 수업 시간 정책

- 수업 운영 시간은 05:00부터 22:00까지입니다.
- 수업 시작 시간은 정시와 30분 단위를 사용합니다.
- 일반 수업 길이는 30분 또는 60분을 지원합니다.
- 회원이 신청할 수 있는 수업 범위는 기본적으로 30일 이내입니다.
- 관리자는 최근 2주 이내 지난 수업도 수업 탭에서 확인할 수 있습니다.

### 5.5 고정수업 정책

고정수업은 회원이 매주 반복해서 듣는 기본 강습입니다.

관리자가 설정하는 값:

- 회원
- 요일
- 시작 시간
- 수업 길이
- 담당 강사
- 상품 인원

관리자는 회원별로 여러 개의 고정수업을 등록할 수 있습니다.

### 5.6 고정수업 취소 정책

회원이 고정수업에 참석하지 못하는 경우, 회원은 해당 수업에 대해 취소 요청을 만듭니다.

정책:

- 본인의 고정수업만 취소 요청할 수 있습니다.
- 수업 취소 요청은 수업 전날까지만 가능합니다.
- 당일 수업은 취소 요청이 마감됩니다.
- 활성 회원권 기준 8회당 2회까지만 취소 요청할 수 있습니다.
- 승인 대기 중인 취소 요청과 승인된 취소 요청을 함께 계산합니다.
- 관리자가 승인하면 해당 고정수업은 취소 처리되고 빈자리로 공개됩니다.
- 다른 회원이 이미 그 빈자리에 배정된 경우 취소 철회가 제한될 수 있습니다.

관리자 처리:

- 승인
- 거절

### 5.7 개별수업 취소 정책

개별수업은 추가수업 또는 자유수영처럼 고정수업이 아닌 방식으로 배정된 수업입니다.

정책:

- 회원은 본인의 승인된 개별수업을 취소할 수 있습니다.
- 개별수업 취소는 수업 전날까지만 가능합니다.
- 당일 수업은 취소할 수 없습니다.
- 횟수권이 차감된 추가수업은 취소 시 횟수권을 복구합니다.
- 자유수영처럼 횟수권 차감이 없던 예약은 예약만 취소합니다.
- 취소된 예약은 `reservations.canceled_at`으로 기록됩니다.

### 5.8 추가수업 신청 정책

추가수업은 열린 수업 중 아직 배정된 회원이 없는 시간에 신청합니다.

정책:

- 회원 계정만 신청할 수 있습니다.
- 남은 횟수권이 있어야 신청할 수 있습니다.
- 이미 배정된 시간에는 신청할 수 없습니다.
- 이미 승인 대기 중인 신청이 있으면 중복 신청할 수 없습니다.
- 관리자가 승인하면 수업권 1회가 차감되고 예약이 생성됩니다.
- 관리자가 거절하면 예약이 생성되지 않습니다.

### 5.9 자유수영 신청 정책

자유수영은 영업시간 안의 닫힌 시간에 신청합니다.

정책:

- 회원 계정만 신청할 수 있습니다.
- 열린 수업에는 자유수영으로 신청할 수 없습니다.
- 닫힌 시간이어야 합니다.
- 관리자가 승인하면 예약이 생성됩니다.
- 자유수영은 추가수업과 다르게 수업권 차감 정책을 별도로 적용하지 않습니다.

### 5.10 수업 변경 정책

현재 회원 수업 탭에서는 신규 수업 변경 버튼을 제공하지 않습니다.

운영 방식:

- 회원이 수업을 바꾸고 싶으면 기존 수업을 취소합니다.
- 이후 원하는 새 수업을 추가수업 또는 자유수영으로 신청합니다.
- 기존에 생성된 변경 요청 데이터가 있다면 요청 관리와 관리자 승인 요청에서 확인 또는 처리할 수 있습니다.

### 5.11 특별수업 정책

특별수업은 이벤트성 또는 단기성 수업입니다.

관리자 등록 정보:

- 제목
- 설명
- 이미지
- 시작 일시
- 담당 강사
- 수업 시간
- 정원

회원 신청:

- 회원은 특별수업에 신청할 수 있습니다.
- 정원과 신청 순서에 따라 대기 상태가 될 수 있습니다.
- 회원은 신청을 취소할 수 있습니다.

관리자 처리:

- 신청자 승인
- 신청자 거절
- 특별수업 수정

### 5.12 상품 구매 정책

상품 구매는 신청 후 관리자 확정 방식입니다.

관리자:

- 상품명 등록
- 설명 등록
- 이미지 등록
- 가격 등록
- 재고 등록

회원:

- 상품 목록 확인
- 구매 신청
- 대기 중인 구매 신청 취소

재고 정책:

- 회원이 구매 신청하면 재고가 차감됩니다.
- 관리자가 구매 신청을 취소하면 재고가 복구됩니다.
- 관리자가 확정하면 구매 신청은 확정 상태가 됩니다.

### 5.13 공지 정책

- 공지는 관리자만 작성, 수정, 삭제할 수 있습니다.
- 회원과 관리자는 공지를 조회할 수 있습니다.
- 공지 이미지는 Supabase Storage에 저장됩니다.

### 5.14 피드백 정책

- 관리자는 종료된 수업의 회원별 피드백을 작성합니다.
- 피드백은 글, 사진, 동영상을 포함할 수 있습니다.
- 회원은 자기 수업 피드백을 확인합니다.
- 피드백 미디어는 비공개 Storage 버킷에 저장하고, 일정 시간 유효한 signed URL로 조회합니다.

### 5.15 문의 정책

- 회원은 내정보 탭에서 문의를 접수합니다.
- 관리자는 문의를 처리중, 해결, 거절 상태로 관리합니다.
- 관리자는 답변 메모를 남길 수 있습니다.

## 8. 권한 및 보안 구조

### 8.1 기본 원칙

- 앱에는 Supabase publishable key만 사용합니다.
- service role key는 앱에 넣지 않습니다.
- DB 비밀번호를 앱에 넣지 않습니다.
- 고권한 작업은 DB 함수 또는 Edge Function에서 처리합니다.
- RLS 정책으로 회원과 관리자의 데이터 접근 범위를 분리합니다.

### 8.2 회원 권한

회원은 다음 데이터 중심으로 접근합니다.

- 본인 프로필
- 본인 고정수업
- 본인 수업 신청
- 본인 취소 요청
- 본인 예약
- 본인 특별수업 신청
- 본인 구매 신청
- 본인 문의
- 본인 피드백
- 전체 공지
- 활성 상품
- 모집 중인 특별수업

### 8.3 관리자 권한

관리자는 운영상 필요한 전체 데이터를 조회하거나 처리합니다.

- 전체 회원 요약
- 전체 수업 슬롯
- 전체 요청
- 전체 문의
- 전체 구매 신청
- 전체 특별수업 신청
- 공지 작성/수정/삭제
- 피드백 작성
- 회원권 조정
- 수업 운영 관리

### 8.4 계정 삭제

계정 삭제는 앱에서 직접 관리자 권한을 사용하지 않고 Supabase Edge Function을 통해 처리합니다.

## 10. 주요 사용자 흐름

### 10.1 고정수업 취소 흐름

1. 회원이 수업 탭에서 본인 고정수업을 확인합니다.
2. 회원이 `취소` 버튼을 누릅니다.
3. 앱이 취소 요청을 생성합니다.
4. 요청 상태는 승인 대기입니다.
5. 관리자가 홈의 승인 요청에서 확인합니다.
6. 관리자가 승인하면 고정수업이 취소 처리됩니다.
7. 해당 수업은 빈자리로 공개됩니다.
8. 다른 회원이 추가수업으로 신청할 수 있습니다.

### 10.2 개별수업 취소 흐름

1. 회원이 승인된 추가수업 또는 자유수영을 확인합니다.
2. 회원이 `취소` 버튼을 누릅니다.
3. 앱이 본인 예약 취소 RPC를 호출합니다.
4. 수업 전날까지인 경우 예약이 취소됩니다.
5. 추가수업처럼 횟수권이 차감된 예약이면 횟수권이 복구됩니다.
6. 수업 당일이면 취소가 제한됩니다.

### 10.3 추가수업 신청 흐름

1. 관리자가 수업을 열어둡니다.
2. 해당 수업에 배정된 회원이 없으면 회원 화면에 추가수업 신청 가능 상태로 표시됩니다.
3. 회원이 `추가신청`을 누릅니다.
4. 신청은 승인 대기 상태가 됩니다.
5. 관리자가 승인하면 예약이 생성되고 횟수권이 차감됩니다.
6. 관리자가 거절하면 예약이 생성되지 않습니다.

### 10.4 자유수영 신청 흐름

1. 영업시간 내 닫힌 시간이 회원 화면에 자유수영 후보로 표시됩니다.
2. 회원이 `자유수영`을 누릅니다.
3. 신청은 승인 대기 상태가 됩니다.
4. 관리자가 승인하면 예약이 생성됩니다.
5. 회원은 승인된 자유수영을 전날까지 취소할 수 있습니다.

### 10.5 상품 구매 흐름

1. 관리자가 상품을 등록합니다.
2. 회원이 상품 구매 메뉴에서 상품을 확인합니다.
3. 회원이 구매 신청을 합니다.
4. 재고가 먼저 차감됩니다.
5. 관리자가 확정 또는 취소합니다.
6. 취소 시 재고가 복구됩니다.

### 10.6 문의 처리 흐름

1. 회원이 내정보에서 문의를 작성합니다.
2. 문의는 관리자 홈의 승인 요청에 표시됩니다.
3. 관리자는 처리중, 해결, 거절 상태를 선택합니다.
4. 관리자는 답변 메모를 남길 수 있습니다.
5. 회원은 문의 상태를 확인합니다.

## 11. 운영 유의사항

- `supabase/schema.sql`을 최신 상태로 Supabase SQL Editor에 실행해야 DB 함수와 정책이 앱과 맞습니다.
- 앱 수정 후에는 `npm run typecheck`로 TypeScript 오류를 확인합니다.
- 미디어 업로드 기능을 쓰려면 Storage 버킷과 정책이 정상 적용되어 있어야 합니다.
- 테스트 계정 전환 버튼은 개발 모드와 `.env` 테스트 계정 설정이 있을 때만 표시됩니다.
- 실제 배포 전에는 개인정보 처리방침, 계정 삭제 안내, 앱스토어 심사 자료, 푸시 권한 정책을 확인해야 합니다.

## 12. 현재 구현 기준 요약

- 회원 신규 변경 버튼은 제거되어 있습니다.
- 수업 변경은 기존 수업 취소 후 새 수업 신청 방식으로 운영합니다.
- 고정수업 취소 요청은 관리자 승인 방식입니다.
- 개별수업 취소는 회원이 직접 취소하는 방식입니다.
- 고정수업 취소 요청은 수업 전날까지만 가능하며, 회원권 8회당 2회 제한이 있습니다.
